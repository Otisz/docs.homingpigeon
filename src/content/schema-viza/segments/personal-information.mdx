# Personal information schema

## Used in
- [Checkin Guest - `manual`](/schema-viza/segments/checkin-guest#manual)
- [Checkin Guest - `scanned`](/schema-viza/segments/checkin-guest#scanned)

## Good to know
`firstName`, `lastName`, `documentType` and `documentNumber` properties are **required** to have value a least in `scanned` or `manual`,

### Scanned
**VIZA expects all fields that the scanner could scan from the document to filled out.**

The scanned data must be filled in without any modification. \
Except the `nationality`, `gender` and `document type`, which have predefined values.

### Manual
The easiest way to handle `manual` properties, is to "copy" the `scanned` properties and allow the PMS user to modify them
and send _that_ object.

We will handle the "duplicated" values correctly, for example:
```json
{
  "manual": {
    "documentNumber": "123456AA"
  },
  "scanned": {
    "documentNumber": "123456AA"
  }
}
```

## Properties
### `firstName`
The first name of the guest.
- Type: `string` | `null`

### `lastName`
The last name of the guest.
- Type: `string` | `null`

### `birthFirstName`
The guest's birth first name.
- Type: `string` | `null`

### `birtLastName`
The guest's birth last name.
- Type: `string` | `null`

### `motherFirstName`
The guest's mother's maiden first name.
- Type: `string` | `null`

### `motherLastName`
The guest's mother's maiden last name.
- Type: `string` | `null`

### `dateOfBirth`
The guest's date of birth.
- Type: `string` | `null`

### `placeOfBirth`
The guest's place of birth.
- Type: `string` | `null`

### `nationality`
The country code of the guest's nationality.
- Type: `enum` | `null`
- Rules: [`oneOf: Country (enum)`](/schema-viza/enums/country)

### `gender`
The guest's gender.
- Type: `enum` | `null`
- Rules: [`oneOf: Gender (enum)`](/schema-viza/enums/gender)

### `documentType`
The type of identification document presented by the guest at check-in.
- Type: `enum` | `null`
- Rules: [`oneOf: Document type (enum)`](/schema-viza/enums/document-type)

### `documentNumber`
The identification document number presented by the guest at check-in.
- Type: `string` | `null`
